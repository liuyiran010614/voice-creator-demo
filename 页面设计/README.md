# 自然语言捏音色 / 生成发音人 — 项目说明

> 一个用自然语言描述音色、一键生成 3 个可选发音人、并登记保存的网页产品（当前为可点击演示原型）。

## 这是什么
用户用一句话描述想要的音色（例：一位年迈的智者，声音略带沙哑但充满智慧），输入一段试听文本，点击「生成音色」即可得到 3 个候选音色，试听后选中并登记保存为发音人。

当前阶段为 **演示原型**：界面完整、流程可走通，试听为视觉反馈（不真实出声），数据保存在浏览器本地。

## 怎么打开预览
直接双击根目录下的 `index.html`，用浏览器（推荐 Chrome / Edge）打开即可。无需安装任何环境、无需联网。

## 文件夹结构与用途
- `index.html` — 最终产品页面，双击即可预览。
- `docs/` — 项目标准文件，规定"做什么、怎么做、长什么样、按什么顺序做"：
  - [docs/01-需求文档.md](docs/01-需求文档.md) — 产品功能与交互规则。
  - [docs/02-技术方案.md](docs/02-技术方案.md) — 技术选型、文件结构、假数据设计。
  - [docs/03-设计规范.md](docs/03-设计规范.md) — UI 规范：百度蓝色板、字体、间距、组件样式。
  - [docs/04-执行步骤.md](docs/04-执行步骤.md) — 分阶段开发步骤与验收标准。
- `开发日志/` — 按日期记录每天完成事项与待办，文件名格式 `YYYY-MM-DD.md`。

## 工作说明（给产品经理）
1. 有新需求或想改交互 → 先更新 [docs/01-需求文档.md](docs/01-需求文档.md)，再让我据此开发。
2. 想调整颜色 / 风格 → 改 [docs/03-设计规范.md](docs/03-设计规范.md)。
3. 想知道进度 → 看 `开发日志/` 里最新日期的文件。
4. 开发按 [docs/04-执行步骤.md](docs/04-执行步骤.md) 分阶段进行，每个阶段做完都会更新当天开发日志。
