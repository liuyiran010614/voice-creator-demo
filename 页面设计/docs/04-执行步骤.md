# 04 - 执行步骤

分阶段稳健推进，每个阶段完成后自测验收并更新当天开发日志。

## 阶段 0：项目骨架与文档
- 创建 `docs/`、`开发日志/` 文件夹与 `README.md`。
- 写入四份 docs 标准文件（需求 / 技术 / 设计 / 执行）。
- 创建当天开发日志。
- 验收：目录结构齐全，文档内容完整可读。

## 阶段 1：页面静态结构 + UI 样式
- 创建 `index.html`，搭出 4 个板块静态布局。
- 套用 03-设计规范 的色板与组件样式。
- 验收：浏览器打开，4 个板块从上到下排版正确、风格统一（暂无交互）。

## 阶段 2：提示词与试听文本联动
- 渲染推荐角色卡片，实现「换一批」随机刷新。
- 点击角色 → 提示词框 + 试听文本框双自动填充。
- 验收：点角色后两框自动填入对应内容，换一批可刷新。

## 阶段 3：生成音色与选择
- 「生成音色」→ loading → 固定返回 3 个音色卡片。
- 每卡可试听（视觉反馈）、单选高亮。
- 「确认选择」按钮：未选灰、选中可点。
- 验收：完整跑通 生成 → 试听 → 选择 → 确认。

## 阶段 4：音色登记弹窗 + 保存
- 「确认选择」弹出登记弹窗，表单含名称 / 头像 / 描述（自动带入）/ 标签。
- 「保存」写入 localStorage，弹成功提示。
- 验收：选中音色后弹窗，填写并保存成功。

## 阶段 5：整体走查与收尾
- 端到端流程自测，修边距 / 文案 / 交互细节。
- 更新 README 进度与最终开发日志。
- 验收：端到端演示无卡点。

## 验收通用标准
- 离线可用（不依赖任何网络资源）。
- 界面符合 03-设计规范。
- 交互符合 01-需求文档 的规则。
