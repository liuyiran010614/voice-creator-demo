# 02 - 技术方案

## 一、技术选型
- 单个 `index.html` 文件，内置 HTML + CSS + JavaScript（原生，无框架）。
- 双击即可在浏览器打开，无需安装环境、无需联网、无需构建。
- 选择理由：交付物最简单，产品经理无需任何技术配置即可预览与分享。

## 二、文件结构
```
页面设计/
├── index.html      # 全部代码内置于此（结构 + 样式 + 脚本）
├── README.md
├── docs/
└── 开发日志/
```
index.html 内部组织：
- `<style>`：设计规范中的色板、组件样式（见 03-设计规范.md）。
- `<body>`：4 个板块的 DOM 结构。
- `<script>`：角色池数据、交互逻辑、localStorage 读写。

## 三、假数据设计
- `ROLES`：角色数组，每项 `{ id, name, emoji, color, prompt, text }`。
- 「换一批」：从 `ROLES` 中随机抽取 N 个（如 4 个）渲染到推荐区。
- 「生成音色」：固定返回 3 个结果对象 `{ id, name, tag }`（如 音色 A/B/C + 特征标签），用 setTimeout 模拟 loading。
- 试听：点击切换播放状态，用 CSS 动画 + 进度条做视觉反馈，无音频。

## 四、数据存储
- 登记保存的发音人写入 `localStorage`，键名如 `voice_persons`，值为 JSON 数组。
- 每条记录：`{ name, avatar, desc, tag, voiceId, createdAt }`。

## 五、兼容与约束
- 目标浏览器：Chrome / Edge 现代版本。
- 全中文界面。
- 不引入任何外部 CDN / 网络资源，确保离线可用。
