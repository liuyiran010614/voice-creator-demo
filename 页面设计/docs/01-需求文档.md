# 01 - 需求文档

## 一、产品定位
「自然语言捏音色 / 生成发音人」网页产品。用户用自然语言描述音色特征 + 输入试听文本，一键生成 3 个可选发音人音色，完成选择后登记保存。

当前交付目标：**可点击演示原型**（单 HTML 文件，试听为视觉反馈，数据存浏览器本地 localStorage）。

## 二、核心流程
描述音色 → 输入试听文本 → 生成 3 个音色 → 试听并选择 1 个 → 登记保存为发音人。

## 三、页面模块与交互规则（从上到下）

### 1. 提示词板块（捏音色描述）
- 主输入框：多行文本，用户输入音色特征描述。
  - placeholder 示例：`一位年迈的智者，声音略带沙哑但充满智慧，语速平缓，讲述着人生的哲理。`
- 快捷角色推荐区：一排卡片，每张带头像（emoji + 纯色圆形占位）+ 角色名称。
- 「换一批」按钮：从角色池中随机刷新展示的推荐角色。
- 交互：点击某角色 = 自动填充主输入框（提示词）+ 自动填充试听文本框（双填充）。

### 2. 试听文本板块
- 多行输入框，用户可自由编辑。
- 联动：选择推荐角色时自动填入该角色匹配的试听文本。

### 3. 生成音色结果板块
- 「生成音色」按钮：点击后经短暂 loading，固定返回 3 个音色卡片。
- 每个音色卡片：名称（如「音色 A」）+ 简短特征标签 + 「试听」按钮。
- 试听：点击出现播放动画 + 进度条（视觉反馈，不真实出声）。
- 单选：卡片可选中并高亮，同时只能选 1 个。
- 「确认选择」按钮：未选中时置灰不可点；选中 1 个后可点击 → 打开登记弹窗。

### 4. 音色登记板块（弹窗）
- 触发：点击「确认选择」后弹出。
- 表单字段：
  - 发音人名称（输入框，必填）
  - 头像 / 图标（从预置图标中选择）
  - 音色描述（自动带入提示词板块的内容，可编辑）
  - 标签 / 分类（如：男声 / 女声 / 童声 / 旁白）
- 「保存」按钮：保存成功后弹出成功提示，数据存入 localStorage。

## 四、预置角色池（提示词 + 试听文本）
每个推荐角色绑定一组「提示词 + 匹配试听文本」，点击时双填充。示例：

- 儿童动画小动物 🐿️
  - 提示词：声音稚嫩尖细，充满童趣和夸张语气
  - 试听文本：哎呀，我的小松果不见了！是不是被大灰狼偷走了？呜呜呜，我要去找我的松果！
- 年迈智者 🧙
  - 提示词：一位年迈的智者，声音略带沙哑但充满智慧，语速平缓，讲述着人生的哲理
  - 试听文本：孩子，人生如河，急流处需稳舵，平缓处莫忘前行。
- 温柔女主播 🎀
  - 提示词：温柔甜美的女声，吐字清晰，亲和力强，适合知识讲解
  - 试听文本：欢迎回到今天的节目，接下来为你带来一段温暖的故事。
- 沉稳男旁白 🎙️
  - 提示词：低沉稳重的男声，富有磁性，节奏沉着，适合纪录片旁白
  - 试听文本：在这片古老的土地上，时间留下了无数不为人知的故事。
- 元气少女 ✨
  - 提示词：活泼明亮的少女声，语速偏快，充满活力和热情
  - 试听文本：大家好呀！今天也要元气满满地开始新的一天哦！
- 磁性大叔 🧔
  - 提示词：成熟低沉的中年男声，慵懒随性，带一点烟嗓质感
  - 试听文本：忙碌了一天，不如泡杯咖啡，听一首老歌，放松一下吧。

（角色池可继续扩充，展示时每次随机取若干个，「换一批」刷新。）

## 五、本期不做（边界）
- 不接入真实语音合成 / TTS 服务，试听不出声。
- 不做后端、不做账号体系，数据仅存浏览器本地。
- 不做发音人列表管理页（保存后仅成功提示）。
